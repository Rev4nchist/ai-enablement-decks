The Starting Point

AI without coordination
is just noise.

Most AI usage today is one-shot — a prompt, an answer, forgotten. No memory. No handoffs. No continuity between strategy and execution.

📂

No Persistent Memory

Each session starts cold. Decisions, context, and history are lost between conversations.

🔀

Strategy / Execution Gap

Ideas from a planning session never automatically reach the person building the thing.

🔁

Manual Re-briefing

Every handoff requires the human to re-explain context that the AI already knew last session.

The fix: treat AI like a team, not a tool. Give each AI a role, a memory layer, and a communication protocol — then wire them together.

The System

Three AI personalities.
One coherent system.

🌙 Eve

Personal Strategic Layer

Lives in Claude.ai. Handles Dave's personal projects — 𝕏 Research, Minions delegations, personal direction-setting.

Claude.ai — Personal
  • Owns Eve→Code handoff queue
  • Reads Code→Eve results
  • Personal decisions & direction
⚡ Donna

Work Strategic Layer

Lives in Claude.ai. Scoped exclusively to Fourth AI work — SPARK, RFPs, competitive intel, AI dashboards, weekly reports.

Claude.ai — Fourth Work
  • Owns Donna→Code handoff queue
  • Reads Code→Donna results
  • Runs the weekly report pipeline
🔨 Code

Execution Layer

Lives in Claude Code (CLI). Shared resource — builds, deploys, and implements what Eve or Donna specify in their handoff queues.

Claude Code — Terminal
  • Receives specs from Eve or Donna
  • Writes results back to Bridge HQ
  • Ships code, reports, deployments

Key Rule

Relations are never set on creation — always a second call. This prevents Notion API conflicts and keeps data clean.

How Work Moves

Handoffs: child pages
+ one-liners in HQ.

Every time Donna passes work to Code, it creates two things: a richly-specified child page and a single-line status entry in the Bridge HQ queue.

The Child Page (Full Spec)
# [Donna→Code] Deploy SPARK v2 — 2026-04-10

From: Donna
Priority: High
Status: 🟡 Pending

## Context
Dave approved v2 scope…

## Spec / Steps
1. Pull latest from main…

## Acceptance Criteria
Dashboard loads under 2s…

## Write Back
Confirm deploy URL…
The HQ One-Liner (Status View)
🟡
[HANDOFF] 2026-04-10 — Deploy SPARK v2 From: Donna | Priority: High | Deploy dashboard to Railway, confirm live URL.
Status Lifecycle
🟡 Pending → 🔵 In Progress → ✅ Done

Status is synced in both places — HQ one-liner emoji AND the child page header. One always reflects the other.

Bridge Sync command — sweep completed items to Archive, refresh Active Context, and surface any unread Code→Donna results. One command, full audit.

How It Works Day-to-Day

Speak the command.
Donna handles the rest.

Task & Bridge Commands
You typeWhat happens
create a task: [name] Task created in Fourth AI bucket, right priority inferred
pass to Code: [spec] Child page created + HQ one-liner inserted
what did Code build Reads all unread Code→Donna items, surfaces in plain English
log decision: [thing] Inserts dated entry at top of Key Decisions Log
bridge sync Full audit, sweep archive, update Active Context
morning routine Check active tasks + any waiting Code→Donna handoffs
Reporting Commands
You typeWhat happens
weekly report Full pipeline — collect, synthesise, confirm, publish + slides
report status Synthesis summary only — no publish
push report Skip to publish — uses synthesis already in session
slides only Generates HTML deck, skips dashboard push

One confirmation gate rule: Donna infers, announces what she's doing, and acts. She only pauses for irreversible actions — publishes, deletes — where a single confirm is enough.
