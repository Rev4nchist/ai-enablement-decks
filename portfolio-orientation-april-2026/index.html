In Progress

Project 04 of 07

Salesforce MCP
Connector

Per-user Salesforce access inside Claude — built on Azure with Microsoft Entra ID identity. This is the proof-of-concept that unlocks every future enterprise connector Fourth builds.

Planned · Build Underway
Why it exists
Most AI tools use a shared service account — one login that sees everything. This connector uses per-user identity so every Salesforce query runs under the user's own credentials. Real audit trails. Permission-aware. Proper governance from day one.
Current Status
Architecture finalised. Token exchange flow via Entra ID confirmed. FastMCP framework selected. IT ticket for Entra ID app creation submitted and marked urgent — this is the critical path item.
Technical Decisions
  • FastMCP — used by 70% of all MCP servers globally
  • Token exchange via Microsoft Entra ID (not JWT / service account)
  • Deployed to Azure — aligns with IT governance
  • BLOCKER: IT Entra ID app creation ticket — everything gates on this
Strategic Significance
Salesforce is Connector #1. Once this pattern is validated, it becomes the template for every future MCP connector — Zendesk, HubSpot, ServiceNow, internal APIs. The architecture decision here compounds.

Project 06 of 07

AI Tool
Standardization

Establishing how Fourth builds, versions, distributes, and governs AI tools — for both developer teams and business users. The governance framework that makes everything else scale.

In Progress · Deadline Apr 30
Why it exists
Without a framework, every team builds their own AI tools their own way — no reuse, no governance, no security standards. This project sets the rules of the road so the AI programme can scale without becoming a liability.
What's Been Decided
  • GitHub as single source of truth for all AI tooling
  • Two-repo architecture: ai-artifactory (engineering) + fourth-ai-toolkit (business)
  • Claude as primary dev platform — port to others as they mature
  • MCP servers hosted in Azure AI Landing Zone
  • Project-level plugin distribution (not enterprise-wide push)
Two Audiences, Two Strategies
Business users (Sales, CS, Marketing) — pre-built tools, no GitHub required. Tools are discoverable and distributed via the Agent-Arch Platform.

Developer users — fine-grained control, tools versioned in GitHub (ai-artifactory), opt-in per project
