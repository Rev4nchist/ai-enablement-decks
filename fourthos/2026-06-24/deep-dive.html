Concept spotlight

Here's the idea
Governance proxy (FastMCP v3)

A connector that sits between an AI agent and a business system like Salesforce. It enforces who you are (your own identity, via Entra then on-behalf-of token exchange), what you can see (read-only, default-deny), and records what happened (full audit) — so AI can act across systems safely under each person's own permissions, instead of through one shared god-key. Build it once and every future connector (Zendesk, NetSuite, Jira) inherits the same identity, audit, and reporting standard.

Architecture

The governance-proxy pattern

Every request flows through one governed checkpoint. The proxy resolves per-user identity, enforces read-only / default-deny access, and writes a unified audit trail — then brokers the call to the live system. Future connectors branch off the same proxy.

Live in production Future connectors (planned)

One governed checkpoint, many systems: the proxy enforces identity, audit, and read-only access before any call reaches a live system — and every future connector inherits the same standard.
